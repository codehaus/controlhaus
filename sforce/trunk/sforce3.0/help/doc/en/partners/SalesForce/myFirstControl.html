Howto use sforce 3.0 Workshop Custom Control
After you have installed sForce 3.0 Workshop Custom control (as an extension) you can use the exposed methods on the control. You have to create a project selecting one project type among Weblogic Workshop project templates or create your own project template. In case the control was installed successfully you must see it among the installed controls in Weblogic Workshop. When you are inserting the control you will be prompted to insert a name for the control. In case this is the first time using the SalesForce wrapper, you will be prompted to download your company specific WSDL file. For this one, you should insert your account name and password - this way the controll will download automatically the wsdl file, create and compile the Schema project and install the Object browser. In case of an error (communication error is most likely) you have to do the following: Log into SalesForce and download manually your company's WSDL file. This is the link from where you can downlaod your WSDL : Get WSDL manually . Save the WSDL file as enterprise.wsdl and copy under your Schema directory. In case you don't have a directory called Schema, create a Schema project, named Scheme with Weblogic Workshop. After the copy, Workshop will startd building the project and will create the propper XMLBeans classes. After this procedure you can try to add the wrapper again but you don't have get the WSDL this time. Since now you can use theexposed methods in the sForce wrapper and access your SalesForce specific data.
